  9 /**
 10   @class
 11   
 12   An SC.Gesture analyzes SC.Touch objects and determines if they are part
 13   of a gesture. If they are, SC.Gestures keep the views that own them up-to-date
 14   as that gesture progresses, informing it when it starts, when some aspect of
 15   it changes, when it ends, and—for convenience—when it is considered to have
 16   been "triggered".
 17   
 18   Gestures can call the following methods on their views:
 19   
 20   - [gestureName](gesture, args...): called when the gesture has occurred. This is 
 21     useful for event-style gestures, where you aren't interested in when it starts or
 22     ends, but just that it has occurred. SC.SwipeGesture triggers this after the
 23     swipe has moved a minimum amount—40px by default.
 24 
 25   - [gestureName]Start(gesture, args...): called when the gesture is first recognized. 
 26     For instance, a swipe gesture may be recognized after the finger has moved a 
 27     minimum distance in a horizontal.
 28     
 29   - [gestureName]Changed(gesture, args...): called when some property of the gesture 
 30     has changed. For instance, this may be called continuously as the user swipes as 
 31     the swipe's distance changes.
 32   
 33   - [gestureName]Cancelled(gesture, args...): called when a gesture, for one reason 
 34     or another, is no longer recognized. For instance, a horizontal swipe gesture 
 35     could cancel if the user moves too far in a vertical direction.
 36   
 37   - [gestureName]End(gesture, args...): called when a gesture ends. A swipe would end
 38     when the user lifts their finger.
 39   
 40   Gesture Lifecycle
 41   ------------------------
 42   Gestures start receiving events when their view—usually mixing in SC.Gesturable—tells it
 43   about activities with "unassigned" touches. "Unassigned" touches are touches that have
 44   not _yet_ been assigned to a gesture.
 45   
 46   The touch becomes "assigned" when the gesture's touchIsInGesture method returns YES.
 47   When a tocuh is assigned to a gesture, the gesture becomes the touch's touch responder;
 48   this means that it will receive a touchStart event (to which it must return YES), and
 49   then, all further touch events will be sent _directly_ to the gesture—the gesture's view
 50   will not receive them at all.
 51   
 52   At any point, the gesture may tell the view that it has started, ended, or changed. In
 53   addition, the gesture may tell the view it has been "triggered." A gesture is not
 54   necessarily "triggered" when it starts and ends; for instance, a swipe gesture might
 55   only be triggered if the swipe moves more than a specified amount. The ability to track
 56   when the gesture has been triggered allows views to easily handle the gesture as its own
 57   event, rather than as the individual events that are part of it.
 58   
 59   If, at some point, the gesture must release the touch back (perhaps the gesture had _thought_
 60   the touch was a part of it, but turned out to be incorrect), the release(touch) method releases
 61   it back to the view.
 62   
 63   Exclusivity
 64   ---------------------------------
 65   The concept described above gives the gestures a way to be either exclusive or inclusive as-needed:
 66   they can choose to take exclusive control of a touch if they think it is theirs, but if they are
 67   not sure, they can wait and see.
 68   
 69   Status Object
 70   ---------------------------------
 71   It is a common need to track some data related to the touch, but without modifying the touch itself.
 72   SC.Gesture is able to keep track of simple hashes for you, mapping them to the SC.Touch object,
 73   so that you can maintain some state related to the touch.
 74   
 75   For instance, you could set status.failed in touchesDragged, if a touch that you previously
 76   thought may have been part of the gesture turned out not to be, and then check for 
 77   status.failed in touchIsInGesture, returning NO if present. This would cause the touch
 78   to never be considered for your gesture again.
 79   
 80   touchIsInGesture is called with the status hash provided in the second argument. You may look
 81   up the status hash for a touch at any time by calling this.statusForTouch(touch).
 82   
 83   
 84   Implementing a Gesture
 85   ---------------------------------
 86   To write a gesture, you would generally implement the following methods:
 87   
 88   - touchIsInGesture: Return YES when the touch is—or is likely enough to be that you
 89     want your gesture to have exclusive control over the touch. You usually do not
 90     perform much gesture logic here—instead, you save it for touchStart, which will
 91     get called after you return YES from this method.
 92   
 93   - touchStart: Return YES to accept control of the touch. If you do not return YES,
 94     your gesture will not receive touchesDragged nor touchEnd events. At this point,
 95     you may (or may not) wish to tell the view that the gesture has started by using the
 96     start(args...) method.
 97     
 98   - touchesDragged: Use this as you would use it in an SC.View to track the touches
 99     assigned to the gesture. At this point, you might want to tell the view that the
100     gesture has updated by using the change(args...) method.
101   
102   - touchEnd: Again, use this like you would in an SC.View to track when touches
103     assigned to the gesture have ended. This is also a potential time to alert the view
104     that the gesture has ended, by using the end(args...) method. Further, this may
105     also be the time to "trigger" the gesture.
106   
107 */
108 SC.Gesture = SC.Object.extend({
109   /**
110     The gesture's name. When calling events on the owning SC.View, this name will
111     be prefixed to the methods. For instance, if the method to be called is
112     'Start', and the gesture's name is 'swipe', SC.Gesture will call 'swipeStart'.
113   */
114   name: "gesture",
115 
116   /**
117     Return YES to take exclusive control over the touch. In addition to the
118     SC.Touch object you may take control of, you are also provided a "status"
119     hash, which is unique for both the gesture instance and the touch instance,
120     which you may use for your own purposes. 
121   */
122   touchIsInGesture: function(touch, status) {
123     return NO;
124   },
125   
126   /**
127     After you return YES from touchIsInGesture (or otherwise 'take' a touch, perhaps
128     using the 'take' method), touchStart will be called.
129     
130     This is where you do any logic needed now that the touch is part of the gesture.
131     For instance, you could inform the view that the gesture has started by calling
132     this.start().
133     
134     NOTE: SC.Gesture is just like SC.View in that it has an acceptsMultitouch property.
135     If NO (the default), the gesture will only receive touchStart for the first touch
136 137     assigned to it, and only receive touchEnd for the last touch that ends.
138   */
139   touchStart: function(touch) {
140     
141   },
142   
143   /**
144     Called when touches assigned to the gesture have moved.
145     
146     This is where you update the gesture's state, potentially calling change() to
147     notify the view.
148 149   */
150   touchesDragged: function(evt, touches) {
151     
152   },
153   
154   /**
155     Called when a touch assigned to the gesture ends.
156     
157     If there are no remaining touches on the gesture, you may want to call end() to
158     notify the view that the gesture has ended (if you haven't ended the gesture
159     already).
160     
161     NOTE: SC.Gesture is just like SC.View in that it has an acceptsMultitouch property.
162     If NO (the default), the gesture will only receive touchStart for the first touch
163     assigned to it, and only receive touchEnd for the last touch that ends.
164   */
165   touchEnd: function(touch) {
166     
167   },
168   
169   /**
170     Starts the gesture (marking it as "active"), and notifies the view.
171     
172     You can pass any number of arguments to start. They will, along with
173     the gesture instance itself, will be passed to the appropriate gesture 
174     event on the SC.View.
175   */
176   start: function() {
177     if (!this.get("isActive")) {
178       this.set("isActive", YES);
179       
180       var args = SC.$A(arguments);
181       args.unshift(this);
182       
183       var act = this.name + "Start";
184       if (this.view[act]) this.view[act].apply(this.view, args);
185     }
186   },
187   
188   /**
189     Ends the gesture, if it is active (marking it as not active), and notifies
190     the view.
191     
192     You may pass any number of arguments to end(). They, along with your gesture
193     instance itself, will be passed to the appropriate gesture event on the SC.View.
194   */
195   end: function() {
196     if (this.get("isActive")) {
197       this.set("isActive", NO);
198 
199       var args = SC.$A(arguments);
200       args.unshift(this);
201       
202       var act = this.name + "End";
203       if (this.view[act]) this.view[act].apply(this.view, args);
204     }
205   },
206   
207   /**
208     If the gesture is active, notifies the view that the gesture has
209     changed.
210     
211     The gesture, along with any arguments to change(), will be passed to
212     the appropriate method on the SC.View.
213   */
214   change: function() {
215     if (this.get('isActive')) {
216       var args = SC.$A(arguments);
217       args.unshift(this);
218 
219       var act = this.name + "Changed";
220       if (this.view[act]) this.view[act].apply(this.view, args);
221     }
222   },
223 
224   /**
225     Cancels the gesture, if it is active, and notifies the view that the
226     gesture has been cancelled.
227     
228     Gestures are cancelled when they have ended, but any action that would
229     normally be appropriate due to their ending should not be performed.
230     
231     The gesture, along with any arguments to cancel(), will be passed to the
232     appropriate method on the SC.View.
233   */
234   cancel: function(){
235     if (this.get('isActive')) {
236       this.set('isActive', NO);
237 
238       var args = SC.$A(arguments);
239       args.unshift(this);
240 
241       var act = this.name + "Cancelled";
242       if (this.view[act]) this.view[act].apply(this.view, args);
243     }
244   },
245   
246   /**
247     Triggers the gesture, notifying the view that the gesture has happened.
248     
249     You should trigger a gesture where it would be natural to say it has "happened";
250     for instance, if a touch moves a couple of pixels, you probably wouldn't say
251     a swipe has occurred—though you might say it has "begun." And you wouldn't necessarily
252     wait until the touch has ended either. Once the touch has moved a certain amount,
253     there has definitely been a swipe. By calling trigger() at this point, you will
254     tell the view that it has occurred.
255     
256     For SC.SwipeGesture, this allows a view to implement only swipe(), and then be 
257     automatically notified whenever any swipe has occurred.
258   */
259   trigger: function() {
260     var args = SC.$A(arguments);
261     args.unshift(this);
262     
263     var act = this.name;
264     if (this.view[act]) this.view[act].apply(this.view, args);
265   },
266 
267   /**
268     Takes possession of a touch. This does not take effect immediately; it takes effect after
269     the run loop finishes to prevent it from being called during another makeTouchResponder.
270     
271     This is called automatically when you return YES from touchIsInGesture.
272   */
273   take: function(touch) {
274     touch.isTaken = YES; // because even changing responder won't prevent it from being used this cycle.
275     if (SC.none(touch.touchResponder) || touch.touchResponder !== this) touch.makeTouchResponder(this, YES);
276   },
277   
278   /**
279     Releases a touch back to its previous owner, which is usually the view. This allows
280     you to give back control of a touch that it turns out is not part of the gesture. 
281     
282     This takes effect immediately, because you would usually call this from
283     touchesDragged or such.
284   */
285   release: function(touch) {
286     touch.isTaken = NO;
287     if (touch.nextTouchResponder) touch.makeTouchResponder(touch.nextTouchResponder);
288   },
289   
290   /**
291     Discards a touch, making its responder null. This makes the touch go away and never
292     come back—not to this gesture, nor to any other, nor to the view, nor to any other
293     view.
294   */
295   discardTouch: function(touch) {
296     touch.isTaken = YES; // because even changing responder won't prevent it from being used this cycle.
297     touch.makeTouchResponder(null);
298   },
299   
300   /**
301     Returns a status hash (which gestures may and should modify) for a given touch, for tracking
302     whether it is a potential match, etc.
303   */
304   statusForTouch: function(touch) {
305     var key = SC.guidFor(touch.view) + this.name;
306     var status = touch[key];
307     if (!status) status = touch[key] = {};
308     return status;
309   },
310   
311   /**
312     Called when an unassigned touch has started. By default, this calls touchIsInGesture, and,
313     if it returns YES, takes possesion of the touch (causing touchStart to  be called).
314   */
315   unassignedTouchDidStart: function(touch) {
316     if (touch.isTaken) return;
317     if (this.touchIsInGesture(touch, this.statusForTouch(touch))) {
318       this.take(touch);
319     }
320   },
321   
322   /**
323     This is called when the unassigned touches (touches not in a gesture) change or move
324     in some way. By default, this calls touchIsInGesture(touch, status) for each touch.
325   */
326   unassignedTouchesDidChange: function(evt, touches) {
327     touches.forEach(function(touch) {
328       if (touch.isTaken) return;
329       if (this.touchIsInGesture(touch, this.statusForTouch(touch))) {
330         this.take(touch);
331       }
332     }, this);
333   },
334   
335   /**
336     This is called when the unassigned touches (touches not in the gesture) have ended.
337     Default does nothing. Some gestures may want to implement this even if they don't implement
338     unassignedTouchesDidChange.
339   */
340   unassignedTouchDidEnd: function(touch) {
341     
342   },
343   
344   /**
345     Marks the touch as "interesting" to this gesture. A view could then check the 'isInteresting'
346     property of the touch to see if any gestures are interested in it, potentially delaying any
347     action of releasing the touch to another view.
348   */
349   interestedInTouch: function(touch) {
350     var status = this.statusForTouch(touch);
351     if (status.isInterested) return;
352     status.isInterested = YES;
353     touch.isInteresting++;
354   },
355   
356   /**
357     Marks the touch as "uninteresting" to this gesture.
358   */
359   uninterestedInTouch: function(touch) {
360     var status = this.statusForTouch(touch);
361     if (!status.isInterested) return;
362     status.isInterested = NO;
363     touch.isInteresting--;
364   }
365 });
