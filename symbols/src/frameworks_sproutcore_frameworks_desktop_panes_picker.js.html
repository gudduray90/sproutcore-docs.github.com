 10 /**
 11   Popular customized picker position rules:
 12   default: initiated just below the anchor.
 13            shift x, y to optimized picker visibility and make sure top-left corner is always visible.
 14   menu :   same as default rule +
 15            default(1, 4, 3) or custom offset below the anchor for default location to fine tuned visual alignment +
 16            enforce min left(7px)/right(8px) padding to the window

 22           perfect bottom (3) > perfect right (0) > perfect left (1) > perfect top (2)

 62 /**
 63   @class
 64 
 65   Display a non-modal pane that automatically repositions around a view so as
 66   to remain visible.
 67 
 68   An `SC.PickerPane` repositions around the view to which it is anchored as the
 69   browser window is resized so as to ensure the pane's content remains visible.
 70   A picker pane is useful for displaying supplementary information and does not
 71   block the user's interaction with other UI elements. Picker panes typically
 72   provide a better user experience than modal panels.
 73 
 74   An `SC.PickerPane` repositions itself according to the optional `preferMatrix`
 75   argument passed in the `.popup()` method call. The `preferMatrix` either
 76   specifies an offset-based arrangement behavior or a position-based arrangement
 77   behavior depending on the `preferType` argument in the `.popup()` call.
 78 
 79   The simplest way to create and display a picker pane:
 80 
 81       SC.PickerPane.create({
 82         layout: { width: 400, height: 200 },
 83         contentView: SC.View.extend({})
 84       }).popup(someView);
 85 
 86   This displays the `SC.PickerPane` anchored to `someView`.
 87 
 88   ## Positioning
 89 
 90   Picker pane positioning can be classified into two broad categories:
 91   offset-based and position-based.
 92 
 93   ### Offset-based
 94 
 95   When `preferType` is unspecified, `SC.PICKER_MENU` or `SC.PICKER_FIXED`, then
 96   the `preferMatrix` array describes the offset that is used to position the
 97   pane below the anchor. The offset is described by an array of three values,
 98   defaulting to `[1, 4, 3]`. The first value controls the x offset and the second
 99   value the y offset. The third value can be `0` (right) or `3` (bottom),
100   controlling whether the origin of the pane is further offset by the width
101   (in the case of 0) or the height (in the case of 3) of the anchor.
102 
103   ### Position-based
104 
105   When `preferType` is `SC.PICKER_POINTER` or `SC.PICKER_MENU_POINTER`, then
106   the `preferMatrix` specifies the sides in the order in which you want the
107   `SC.PickerPane` to try to arrange itself around the view to which it is
108   anchored. The fifth element in the `preferMatrix` specifies which side the
109   `SC.PickerPane` should display on when there isn't enough space around any
110   of the preferred sides.
111 
112   Anchor sides are defined by their index in `SC.POINTER_LAYOUT`, where right
113   is `0`, left is `1`, top is `2`, and bottom is `3`.
114 
115   For example, the `preferMatrix` of `[3, 0, 1, 2, 2]` says: "Display below the
116   anchor (3); if there isn't enough space then display to the right of the anchor (0).
117   If there isn't enough space either below or to the right of the anchor, then appear
118   to the left (1), unless there is also no space on the left, in which case display
119   above the anchor (2)."
120 
121   ## Position Rules
122 
123   When invoking `.popup()` you can optionally specify a picker position rule with
124   the `preferType` argument.
125 
126   If no `preferType` is specified, the picker pane is displayed just below the anchor.
127   The pane will reposition automatically for optimal visibility, ensuring the top-left
128   corner is visible.
129 
130   These position rules have the following behaviors:
131 
132   ### `SC.PICKER_MENU`
133 
134   Positioning is offset-based, with `preferMatrix` defaulting to `[1, 4, 3]`.
135   Furthermore, a minimum left and right padding to window, of 7px and 8px, respectively,
136   is enforced.
137 
138 
139   ### `SC.PICKER_FIXED`
140 
141   Positioning is offset-based, with `preferMatrix` defaulting to `[1, 4, 3]` and
142   skipping `fitPositionToScreen`.
143 
144 
145   ### `SC.PICKER_POINTER`
146 
147   Positioning is position-based, with `preferMatrix` defaulting to `[0, 1, 2, 3, 2]`,
148   i.e. right > left > top > bottom; fallback to top.
149 
150 
151   ### `SC.PICKER_MENU_POINTER`
152 
153   Positioning is position-based, with `preferMatrix` defaulting to `[3, 0, 1, 2, 3]`,
154   i.e. bottom, right, left, top; fallback to bottom.
155 
156 
157 
158   ## Examples
159 
160   Examples for applying popular customized picker position rules:
161 
162   ### default:
163 
164       SC.PickerPane.create({
165         layout: { width: 400, height: 200 },
166         contentView: SC.View.extend({})
167       }).popup(anchor);
168 
169   ### menu below the anchor with default `preferMatrix` of `[1, 4, 3]`:
170 
171       SC.PickerPane.create({
172         layout: { width: 400, height: 200 },
173         contentView: SC.View.extend({})
174       }).popup(anchor, SC.PICKER_MENU);
175 
176   ### menu on the right side of anchor with custom `preferMatrix` of `[2, 6, 0]`:
177 
178       SC.PickerPane.create({
179         layout: { width: 400, height: 200 },
180         contentView: SC.View.extend({})
181       }).popup(anchor, SC.PICKER_MENU, [2, 6, 0]);
182 
183   ### fixed below the anchor with default `preferMatrix` of `[1, 4, 3]`:
184 
185       SC.PickerPane.create({
186         layout: { width: 400, height: 200 },
187         contentView: SC.View.extend({})
188       }).popup(anchor, SC.PICKER_FIXED);
189 
190   ### fixed on the right side of anchor with `preferMatrix` of `[-22,-17, 0]`:
191 
192       SC.PickerPane.create({
193         layout: { width: 400, height: 200 },
194         contentView: SC.View.extend({})
195       }).popup(anchor, SC.PICKER_FIXED, [-22,-17, 0]);
196 
197   ### pointer with default `preferMatrix` of `[0, 1, 2, 3, 2]`:
198 
199       SC.PickerPane.create({
200         layout: { width: 400, height: 200 },
201         contentView: SC.View.extend({})
202       }).popup(anchor, SC.PICKER_POINTER);
203 
204   Positioning: right (0) > left (1) > top (2) > bottom (3). Fallback to top (2).
205 
206   ### pointer with custom `preferMatrix` of `[3, 0, 1, 2, 2]`:
207 
208       SC.PickerPane.create({
209         layout: { width: 400, height: 200 },
210         contentView: SC.View.extend({})
211       }).popup(anchor, SC.PICKER_POINTER, [3, 0, 1, 2, 2]);
212 
213   Positioning: bottom (3) > right (0) > left (1) > top (2). Fallback to top (2).
214 
215   ### menu-pointer with default `preferMatrix` of `[3, 0, 1, 2, 3]`:
216 
217       SC.PickerPane.create({
218         layout: { width: 400, height: 200 },
219         contentView: SC.View.extend({})
220       }).popup(anchor, SC.PICKER_MENU_POINTER);
221 
222   Positioning: bottom (3) > right (0) > left (1) > top (2). Fallback to bottom (3).

436   /** @private
437     The ideal position for a picker pane is just below the anchor that
438     triggered it + offset of specific preferType. Find that ideal position,
439     then call fitPositionToScreen to get final position. If anchor is missing,
440     fallback to center.
441   */
442   positionPane: function (useAnchorCached) {
443     useAnchorCached = useAnchorCached && this.get('anchorCached');
444 
445     var anchor       = useAnchorCached ? this.get('anchorCached') : this.get('anchorElement'),
446         frame        = this.get('borderFrame'),
447         preferType   = this.get('preferType'),
448         preferMatrix = this.get('preferMatrix'),
449         layout       = this.get('layout'),
450         origin;
451 
452     // usually an anchorElement will be passed.  The ideal position is just
453     // below the anchor + default or custom offset according to preferType.
454     // If that is not possible, fitPositionToScreen will take care of that for
455     // other alternative and fallback position.
456 
457     if (anchor) {
458       if (!useAnchorCached) {
459         anchor = this.computeAnchorRect(anchor);
460         this.set('anchorCached', anchor);
461       }
462 
463       origin = SC.cloneRect(anchor);
464 
465       if (preferType) {
466         switch (preferType) {
467         case SC.PICKER_MENU:
468         case SC.PICKER_FIXED:
469           if (!preferMatrix || preferMatrix.length !== 3) {
470             // default below the anchor with fine-tuned visual alignment
471             // for Menu to appear just below the anchorElement.
472             this.set('preferMatrix', [1, 4, 3]);
473           }
474 
475           // fine-tuned visual alignment from preferMatrix
476           origin.x += ((this.preferMatrix[2] === 0) ? origin.width : 0) + this.preferMatrix[0];
477           origin.y += ((this.preferMatrix[2] === 3) ? origin.height : 0) + this.preferMatrix[1];
478           break;
479         default:
480           origin.y += origin.height;
481           break;
482         }
483       } else {
484         origin.y += origin.height;
485       }
486 
487       // Since we repeatedly need to know the half-width and half-height of the
488       // frames, add those properties.
489       anchor.halfWidth = parseInt(anchor.width * 0.5, 0);
490       anchor.halfHeight = parseInt(anchor.height * 0.5, 0);
491       frame.halfWidth = parseInt(frame.width * 0.5, 0);
492       frame.halfHeight = parseInt(frame.height * 0.5, 0);
493 
494       origin = this.fitPositionToScreen(origin, frame, anchor);
495 
496       this.adjust({
497         width: origin.width,
498         height: origin.height,
499         left: origin.x,
500         top: origin.y
501       });
502     // if no anchor view has been set for some reason, just center.
503     } else {
504       this.adjust({
505         width: layout.width,
506         height: layout.height,
507         centerX: 0,
508         centerY: 0
509       });
510     }
511     this.updateLayout();
512 
513     return this;
514   },

516   /** @private
517     This method will return ret (x, y, width, height) from a rectangular element
518     Notice: temp hack for calculating visible anchor height by counting height
519     up to window bottom only. We do have 'clippingFrame' supported from view.
520     But since our anchor can be element, we use this solution for now.
521   */
522   computeAnchorRect: function (anchor) {
523     var bounding, ret, cq,
524         wsize = SC.RootResponder.responder.computeWindowSize();
525     // Some browsers natively implement getBoundingClientRect, so if it's
526     // available we'll use it for speed.
527     if (anchor.getBoundingClientRect) {
528       // Webkit and Firefox 3.5 will get everything they need by
529       // calling getBoundingClientRect()
530       bounding = anchor.getBoundingClientRect();
531       ret = {
532         x:      bounding.left,
533         y:      bounding.top,
534         width:  bounding.width,
535         height: bounding.height
536       };
537       // If width and height are undefined this means we are in IE or FF < 3.5
538       // if we did not get the frame dimensions the do the calculations
539       // based on an element
540       if (ret.width === undefined || ret.height === undefined) {
541         cq = SC.$(anchor);
542         ret.width = cq.outerWidth();
543         ret.height = cq.outerHeight();
544       }
545     } else {
546       // Only really old versions will have to go through this code path.
547       ret   = SC.offset(anchor); // get x & y
548       cq    = SC.$(anchor);
549       ret.width = cq.outerWidth();
550       ret.height = cq.outerHeight();
551     }
552     ret.height = (wsize.height - ret.y) < ret.height ? (wsize.height - ret.y) : ret.height;
553 
554     if (!SC.browser.isIE && window.scrollX > 0 || window.scrollY > 0) {
555       ret.x += window.scrollX;
556       ret.y += window.scrollY;
557     } else if (SC.browser.isIE && (document.documentElement.scrollTop > 0 || document.documentElement.scrollLeft > 0)) {
558       ret.x += document.documentElement.scrollLeft;
559       ret.y += document.documentElement.scrollTop;
560     }
561     return ret;
562   },

564   /** @private
565     This method will dispatch to the right re-position rule according to preferType
566   */
567   fitPositionToScreen: function (preferredPosition, picker, anchor) {
568     var wsize = SC.RootResponder.responder.computeWindowSize(),
569         wret = { x: 0, y: 0, width: wsize.width, height: wsize.height };
570 
571     // if window size is smaller than the minimum size of app, use minimum size.
572     var mainPane = SC.RootResponder.responder.mainPane;
573     if (mainPane) {
574       var minWidth = mainPane.layout.minWidth,
575           minHeight = mainPane.layout.minHeight;
576 
577       if (minWidth && wret.width < minWidth) {
578         wret.width = mainPane.layout.minWidth;
579       }
580 
581       if (minHeight && wret.height < minHeight) {
582         wret.height = mainPane.layout.minHeight;
583       }
584     }
585 
586     picker.x = preferredPosition.x;
587     picker.y = preferredPosition.y;
588 
589     if (this.preferType) {
590       switch (this.preferType) {
591       case SC.PICKER_MENU:
592         // apply menu re-position rule
593         picker = this.fitPositionToScreenMenu(wret, picker, this.get('isSubMenu'));
594         break;
595       case SC.PICKER_MENU_POINTER:
596         this.setupPointer(anchor);
597         picker = this.fitPositionToScreenMenuPointer(wret, picker, anchor);
598         break;
599       case SC.PICKER_POINTER:
600         // apply pointer re-position rule
601         this.setupPointer(anchor);
602         picker = this.fitPositionToScreenPointer(wret, picker, anchor);
603         break;
604       case SC.PICKER_FIXED:
605         // skip fitPositionToScreen
606         break;
607       default:
608         break;
609       }
610     } else {
611       // apply default re-position rule
612       picker = this.fitPositionToScreenDefault(wret, picker, anchor);
613     }
614     // this.displayDidChange();
615     return picker;
616   },

618   /** @private
619     re-position rule migrated from old SC.OverlayPaneView.
620     shift x, y to optimized picker visibility and make sure top-left corner is always visible.
621   */
622   fitPositionToScreenDefault: function (w, f, a) {
623     var mx;
624 
625     // make sure the right edge fits on the screen.  If not, anchor to
626     // right edge of anchor or right edge of window, whichever is closer.
627     if (SC.maxX(f) > w.width) {
628       mx = Math.max(SC.maxX(a), f.width);
629       f.x = Math.min(mx, w.width) - f.width;
630     }
631 
632     // if the left edge is off of the screen, try to position at left edge
633     // of anchor.  If that pushes right edge off screen, shift back until
634     // right is on screen or left = 0
635     if (SC.minX(f) < 0) {
636       f.x = SC.minX(Math.max(a, 0));
637       if (SC.maxX(f) > w.width) {
638         f.x = Math.max(0, w.width - f.width);
639       }
640     }
641 
642     // make sure bottom edge fits on screen.  If not, try to anchor to top
643     // of anchor or bottom edge of screen.
644     if (SC.maxY(f) > w.height) {
645       mx = Math.max((a.y - f.height), 0);
646       if (mx > w.height) {
647         f.y = Math.max(0, w.height - f.height);
648       } else { f.y = mx; }
649     }
650 
651     // if Top edge is off screen, try to anchor to bottom of anchor. If that
652     // pushes off bottom edge, shift up until it is back on screen or top =0
653     if (SC.minY(f) < 0) {
654       mx = Math.min(SC.maxY(a), (w.height - a.height));
655       f.y = Math.max(mx, 0);
656     }
657     return f;
658   },

660   /** @private
661     Reposition the pane in a way that is optimized for menus.
662 
663     Specifically, we want to ensure that the pane is at least 7 pixels from
664     the left side of the screen, and 20 pixels from the right side.
665 
666     If the menu is a submenu, we also want to reposition the pane to the left
667     of the parent menu if it would otherwise exceed the width of the viewport.
668   */
669   fitPositionToScreenMenu: function (windowFrame, paneFrame, subMenu) {
670     // Set up init location for submenu
671     if (subMenu) {
672       paneFrame.x -= this.get('submenuOffsetX');
673       paneFrame.y -= Math.floor(this.get('menuHeightPadding') / 2);
674     }
675 
676     // If the right edge of the pane is within 20 pixels of the right edge
677     // of the window, we need to reposition it.
678     if ((paneFrame.x + paneFrame.width) > (windowFrame.width - 20)) {
679       if (subMenu) {
680         // Submenus should be re-anchored to the left of the parent menu
681         paneFrame.x = paneFrame.x - (paneFrame.width * 2);
682       } else {
683         // Otherwise, just position the pane 20 pixels from the right edge
684         paneFrame.x = windowFrame.width - paneFrame.width - 20;
685       }
686     }
687 
688     // Make sure we are at least 7 pixels from the left edge of the screen.
689     if (paneFrame.x < 7) { paneFrame.x = 7; }
690 
691     if (paneFrame.y < 7) {
692       paneFrame.height += paneFrame.y;
693       paneFrame.y = 7;
694     }

697     if (paneFrame.height + paneFrame.y + 35 >= windowFrame.height) {

699         paneFrame.y = SC.MenuPane.VERTICAL_OFFSET;
700         paneFrame.height = windowFrame.height - (SC.MenuPane.VERTICAL_OFFSET * 2);
701       } else {
702         paneFrame.y += (windowFrame.height - (paneFrame.height + paneFrame.y + 35));
703       }
704     }
705 
706     return paneFrame;

729   /** @private
730     re-position rule for triangle pointer picker.
731   */
732   fitPositionToScreenPointer: function (w, f, a) {
733     var curType,
734         deltas,
735         matrix = this.preferMatrix,
736         offset = this.pointerOffset,
737         topLefts, botRights,
738         windowPadding = this.get('windowPadding');
739 
740     // Determine the top-left corner of each of the 4 perfectly positioned
741     // frames, while taking the pointer offset into account.
742     topLefts = [
743       // Top left [x, y] if positioned evenly to the right of the anchor
744       [a.x + a.width + offset[0], a.y + a.halfHeight - f.halfHeight],
745 
746       // Top left [x, y] if positioned evenly to the left of the anchor
747       [a.x - f.width + offset[1], a.y + a.halfHeight - f.halfHeight],
748 
749       // Top left [x, y] if positioned evenly above the anchor
750       [a.x + a.halfWidth - f.halfWidth, a.y - f.height + offset[2]],
751 
752       // Top left [x, y] if positioned evenly below the anchor
753       [a.x + a.halfWidth - f.halfWidth, a.y + a.height + offset[3]]
754     ];
755 
756     // Determine the bottom-right corner of each of the 4 perfectly positioned
757     // frames, while taking the pointer offset into account.
758     botRights = [
759       // Bottom right [x, y] if positioned evenly to the right of the anchor
760       [a.x + a.width + f.width + offset[0], a.y + a.halfHeight + f.halfHeight],
761 
762       // Bottom right [x, y] if positioned evenly to the left of the anchor
763       [a.x + offset[1], a.y + a.halfHeight + f.halfHeight],
764 
765       // Bottom right [x, y] if positioned evenly above the anchor
766       [a.x + a.halfWidth + f.halfWidth, a.y + offset[2]],
767 
768       // Bottom right [x, y] if positioned evenly below the anchor
769       [a.x + a.halfWidth + f.halfWidth, a.y + a.height + f.height + offset[3]]
770     ];
771 
772     // Loop through the preferred matrix, hopefully finding one that will fit
773     // perfectly.
774     for (var i = 0, pointerLen = SC.POINTER_LAYOUT.length; i < pointerLen; i++) {
775       // The current preferred side.
776       curType = matrix[i];
777 
778       // Determine if any of the sides of the pane would go beyond the window's
779       // edge for each of the 4 perfectly positioned frames; taking the amount
780       // of windowPadding into account.  This is done by measuring the distance
781       // from each side of the frame to the side of the window.  If the distance
782       // is negative then the edge is overlapping.
783       //
784       // If a perfect position has no overlapping edges, then it is a viable
785       // option for positioning.
786       deltas = {
787         top: topLefts[curType][1] - windowPadding,
788         right: w.width - windowPadding - botRights[curType][0],
789         bottom: w.height - windowPadding - botRights[curType][1],
790         left: topLefts[curType][0] - windowPadding
791       };
792 
793       // UNUSED.  It would be nice to get the picker as close as possible.
794       // Cache the fallback deltas.
795       // if (curType === matrix[4]) {
796       //   fallbackDeltas = deltas;
797       // }
798 
799       // If no edges overflow, then use this layout.
800       if (deltas.top >= 0 &&
801           deltas.right >= 0 &&
802           deltas.bottom >= 0 &&
803           deltas.left >= 0) {
804 
805         f.x = topLefts[curType][0];
806         f.y = topLefts[curType][1];
807 
808         this.set('pointerPosX', 0);
809         this.set('pointerPosY', 0);
810         this.set('pointerPos', SC.POINTER_LAYOUT[curType]);
811 
812         break;
813 
814       // If we prefer right or left and can fit right or left respectively, but
815       // can't fit the top within the window top and padding, then check if by
816       // adjusting the top of the pane down if it would still be beside the
817       // anchor and still above the bottom of the window with padding.
818       } else if (((curType === 0 && deltas.right >= 0) || // Right fits for preferred right
819                  (curType === 1 &&  deltas.left >= 0)) && // or left fits for preferred left,
820                  deltas.top < 0 && // but top doesn't fit,
821                  deltas.top + f.halfHeight >= 0) {  // yet it could.
822 
823         // Adjust the pane position by the amount of downward shifting.
824         f.x = topLefts[curType][0];
825         f.y = topLefts[curType][1] - deltas.top;
826 
827         // Offset the pointer position by the opposite amount of downward
828         // shifting (minus half the height of the pointer).
829         this.set('pointerPosX', 0);
830         this.set('pointerPosY', deltas.top);
831         this.set('pointerPos', SC.POINTER_LAYOUT[curType]);
832         break;
833 
834       // If we prefer right or left and can fit right or left respectively, but
835       // can't fit the bottom within the window bottom and padding, then check
836       // if by adjusting the top of the pane up if it would still be beside the
837       // anchor and still below the top of the window with padding.
838       } else if (((curType === 0 && deltas.right >= 0) || // Right fits for preferred right
839                  (curType === 1 &&  deltas.left >= 0)) && // or left fits for preferred left,
840                  deltas.bottom < 0 && // but bottom doesn't fit,
841                  deltas.bottom + f.halfHeight >= 0) {  // yet it could.
842 
843         // Adjust the pane position by the amount of upward shifting.
844         f.x = topLefts[curType][0];
845         f.y = topLefts[curType][1] + deltas.bottom;
846 
847         // Offset the pointer position by the opposite amount of upward
848         // shifting (minus half the height of the pointer).
849         this.set('pointerPosX', 0);
850         this.set('pointerPosY', Math.abs(deltas.bottom));
851         this.set('pointerPos', SC.POINTER_LAYOUT[curType]);
852         break;
853 
854       // If we prefer top or bottom and can fit top or bottom respectively, but
855       // can't fit the right side within the window right side plus padding,
856       // then check if by adjusting the pane leftwards to fit if it would still
857       // be beside the anchor and still fit within the left side of the window
858       // with padding.
859       } else if (((curType === 2 && deltas.top >= 0) || // Top fits for preferred top
860                  (curType === 3 &&  deltas.bottom >= 0)) && // or bottom fits for preferred bottom,
861                  deltas.right < 0 && // but right doesn't fit,
862                  deltas.right + f.halfWidth >= 0) {  // yet it could.
863 
864         // Adjust the pane position by the amount of leftward shifting.
865         f.x = topLefts[curType][0] + deltas.right;
866         f.y = topLefts[curType][1];
867 
868         // Offset the pointer position by the opposite amount of leftward
869         // shifting (minus half the width of the pointer).
870         this.set('pointerPosX', Math.abs(deltas.right));
871         this.set('pointerPosY', 0);
872         this.set('pointerPos', SC.POINTER_LAYOUT[curType]);
873         break;
874 
875       // If we prefer top or bottom and can fit top or bottom respectively, but
876       // can't fit the left side within the window left side plus padding,
877       // then check if by adjusting the pane rightwards to fit if it would still
878       // be beside the anchor and still fit within the right side of the window
879       // with padding.
880       } else if (((curType === 2 && deltas.top >= 0) || // Top fits for preferred top
881                  (curType === 3 &&  deltas.bottom >= 0)) && // or bottom fits for preferred bottom,
882                  deltas.left < 0 && // but left doesn't fit,
883                  deltas.left + f.halfWidth >= 0) {  // yet it could.
884 
885         // Adjust the pane position by the amount of leftward shifting.
886         f.x = topLefts[curType][0] - deltas.left;
887         f.y = topLefts[curType][1];
888 
889         // Offset the pointer position by the opposite amount of leftward
890         // shifting (minus half the width of the pointer).
891         this.set('pointerPosX', deltas.left);
892         this.set('pointerPosY', 0);
893         this.set('pointerPos', SC.POINTER_LAYOUT[curType]);
894         break;
895       }
896 
897     }
898 
899     // If no arrangement was found to fit, then use the fall back preferred
900     // type.
901     if (i === pointerLen) {
902       if (matrix[4] === -1) {
903         //f.x = a.x > 0 ? a.x + 23 : 0; // another alternative align to left
904         f.x = a.x + a.halfWidth;
905         f.y = a.y + a.halfHeight - f.halfHeight;
906 
907         this.set('pointerPos', SC.POINTER_LAYOUT[0] + ' fallback');
908         this.set('pointerPosY', f.halfHeight - 40);
909       } else {
910         f.x = topLefts[matrix[4]][0];
911         f.y = topLefts[matrix[4]][1];
912 
913         this.set('pointerPos', SC.POINTER_LAYOUT[matrix[4]]);
914         this.set('pointerPosY', 0);
915       }
916 
917       this.set('pointerPosX', 0);
918     }
919 
920     this.invokeLast(this._adjustPointerPosition);
921 
922     return f;
923   },

925   /** @private Measure the pointer element and adjust it by the determined offset. */
926   _adjustPointerPosition: function () {
927     var pointer = this.$('.sc-pointer'),
928       pointerPos = this.get('pointerPos'),
929       marginLeft,
930       marginTop;
931 
932     switch (pointerPos) {
933     case 'perfectRight':
934     case 'perfectLeft':
935       marginTop = -Math.round(pointer.outerHeight() / 2);
936       marginTop += this.get('pointerPosY');
937       pointer.attr('style', "margin-top: " + marginTop + "px");
938       break;
939     case 'perfectTop':
940     case 'perfectBottom':
941       marginLeft = -Math.round(pointer.outerWidth() / 2);
942       marginLeft += this.get('pointerPosX');
943       pointer.attr('style', "margin-left: " + marginLeft + "px;");
944       break;
945     }
946   },

948   /** @private
949     This method will set up pointerOffset and preferMatrix according to type
950     and size if not provided explicitly.
951   */
952   setupPointer: function (a) {
953     var pointerOffset = this.pointerOffset,
954         K             = SC.PickerPane;
955 
956     // set up pointerOffset according to type and size if not provided explicitly
957     if (!pointerOffset || pointerOffset.length !== 4) {
958       if (this.get('preferType') == SC.PICKER_MENU_POINTER) {
959         switch (this.get('controlSize')) {
960         case SC.TINY_CONTROL_SIZE:
961           this.set('pointerOffset', K.TINY_PICKER_MENU_POINTER_OFFSET);
962           this.set('windowPadding', K.TINY_MENU_WINDOW_PADDING);
963           break;
964         case SC.SMALL_CONTROL_SIZE:
965           this.set('pointerOffset', K.SMALL_PICKER_MENU_POINTER_OFFSET);
966           this.set('windowPadding', K.SMALL_MENU_WINDOW_PADDING);
967           break;
968         case SC.REGULAR_CONTROL_SIZE:
969           this.set('pointerOffset', K.REGULAR_PICKER_MENU_POINTER_OFFSET);
970           this.set('windowPadding', K.REGULAR_MENU_WINDOW_PADDING);
971           break;
972         case SC.LARGE_CONTROL_SIZE:
973           this.set('pointerOffset', K.LARGE_PICKER_MENU_POINTER_OFFSET);
974           this.set('windowPadding', K.LARGE_MENU_WINDOW_PADDING);
975           break;
976         case SC.HUGE_CONTROL_SIZE:
977           this.set('pointerOffset', K.HUGE_PICKER_MENU_POINTER_OFFSET);
978           this.set('windowPadding', K.HUGE_MENU_WINDOW_PADDING);
979           break;
980         default:
981           this.set('pointerOffset', K.REGULAR_PICKER_MENU_POINTER_OFFSET);
982           this.set('windowPadding', K.REGULAR_MENU_WINDOW_PADDING);
983           //@if(debug)
984           SC.warn('SC.PickerPane with preferType of SC.PICKER_MENU_POINTER should either define a controlSize or provide a pointerOffset. SC.PickerPane will fall back to default pointerOffset of SC.PickerPane.REGULAR_PICKER_MENU_POINTER_OFFSET and default windowPadding of SC.PickerPane.WINDOW_PADDING');
985           //@endif
986         }
987       } else {
988         var overlapTuningX = (a.width < 16)  ? ((a.width < 4)  ? 9 : 6) : 0,
989             overlapTuningY = (a.height < 16) ? ((a.height < 4) ? 9 : 6) : 0,
990             offsetKey      = K.PICKER_POINTER_OFFSET;
991 
992         var offset = [offsetKey[0] + overlapTuningX,
993                       offsetKey[1] - overlapTuningX,
994                       offsetKey[2] - overlapTuningY,
995                       offsetKey[3] + overlapTuningY];
996 
997         this.set('pointerOffset', offset);
998         // TODO: What to do about this?
999         if (SC.none(this.get('windowPadding'))) {
1000           this.set('windowPadding', K.WINDOW_PADDING);
1001         }
1002       }
1003     } else {
1004       // TODO: What to do about this?
1005       if (SC.none(this.get('windowPadding'))) {
1006         this.set('windowPadding', K.WINDOW_PADDING);
1007       }
1008     }
1009 
1010     // set up preferMatrix according to type if not provided explicitly:
1011     // take default [0, 1, 2, 3, 2] for picker, [3, 0, 1, 2, 3] for menu picker if
1012     // custom matrix not provided explicitly
1013     if (!this.preferMatrix || this.preferMatrix.length !== 5) {
1014       // menu-picker default re-position rule :
1015       // perfect bottom (3) > perfect right (0) > perfect left (1) > perfect top (2)
1016       // fallback to perfect bottom (3)
1017       // picker default re-position rule :
1018       // perfect right (0) > perfect left (1) > perfect top (2) > perfect bottom (3)
1019       // fallback to perfect top (2)
1020       this.set('preferMatrix', this.get('preferType') === SC.PICKER_MENU_POINTER ? [3, 2, 1, 0, 3] : [0, 1, 2, 3, 2]);
1021     }
1022   },
