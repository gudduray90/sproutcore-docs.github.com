 60 /**
 61   @class
 62 
 63   A binding simply connects the properties of two objects so that whenever the
 64   value of one property changes, the other property will be changed also.  You
 65   do not usually work with Binding objects directly but instead describe
 66   bindings in your class definition using something like:
 67 
 68         valueBinding: "MyApp.someController.title"
 69 
 70   This will create a binding from "MyApp.someController.title" to the "value"
 71   property of your object instance automatically.  Now the two values will be
 72   kept in sync.
 73 
 74   One-Way Bindings
 75   ===
 76 
 77   By default, bindings are set up as two-way. In cases where you only need the
 78   binding to function in one direction, for example if a value from a controller
 79   is bound into a read-only LabelView, then for performance reasons you should
 80   use a one-way binding. To do this, call the very useful `oneWay` helper:
 81 
 82       valueBinding: SC.Binding.oneWay('MyApp.someController.title')
 83 
 84   or:
 85 
 86       valueBinding: SC.Binding.from('MyApp.someController.title').oneWay()
 87 
 88   This way if the value of MyApp.someController.title changes, your object's
 89   `value` will also update. Since `value` will never update on its own, this will
 90   avoid the setup time required to plumb the binding in the other direction,
 91   nearly doubling performance for this binding.
 92 
 93   Transforms
 94   ===
 95 
 96   In addition to synchronizing values, bindings can also perform some basic
 97   transforms on values.  These transforms can help to make sure the data fed
 98   into one object always meets the expectations of that object, regardless of
 99   what the other object outputs.
100 
101   To customize a binding, you can use one of the many helper methods defined
102   on SC.Binding. For example:
103 
104         valueBinding: SC.Binding.single("MyApp.someController.title")
105 
106   This will create a binding just like the example above, except that now the
107   binding will convert the value of MyApp.someController.title to a single
108   object (removing any arrays) before applying it to the "value" property of
109   your object.
110 
111   You can also chain helper methods to build custom bindings like so:
112 
113         valueBinding: SC.Binding.single("MyApp.someController.title").notEmpty(null,"(EMPTY)")
114 
115   This will force the value of MyApp.someController.title to be a single value
116   and then check to see if the value is "empty" (null, undefined, empty array,
117   or an empty string).  If it is empty, the value will be set to the string
118   "(EMPTY)".
119 
120   The following transform helper methods are included: `noError`, `single`, `notEmpty`,
121   `notNull`, `multiple`, `bool`, `not`, `isNull`, `and` (two values only), `or` (two
122   values only), and `equalTo`. See each method's documentation for a full description.
123 
124   (Note that transforms are only applied in the forward direction (the 'to' side); values
125   are applied untransformed to the 'from' side. If the 'from' object has validation
126   needs, it should own and apply them itself, for example via a read/write calculated
127   property.)
128 
129   Adding Custom Transforms
130   ===
131 
132   In addition to using the standard helpers provided by SproutCore, you can
133   also defined your own custom transform functions which will be used to
134   convert the value.  To do this, just define your transform function and add
135   it to the binding with the transform() helper.  The following example will
136   not allow Integers less than ten.  Note that it checks the value of the
137   bindings and allows all other values to pass:
138 
139         valueBinding: SC.Binding.transform(function (value, binding) {
140           return ((SC.typeOf(value) === SC.T_NUMBER) && (value < 10)) ? 10 : value;
141         }).from("MyApp.someController.value")
142 
143   If you would like to instead use this transform on a number of bindings,
144   you can also optionally add your own helper method to SC.Binding.  This
145   method should simply return the value of this.transform(). The example
146   below adds a new helper called notLessThan() which will limit the value to
147   be not less than the passed minimum:
148 
149       SC.Binding.notLessThan = function (minValue) {
150         return this.transform(function (value, binding) {
151           return ((SC.typeOf(value) === SC.T_NUMBER) && (value < minValue)) ? minValue : value;
152         });
153       };
154 
155   You could specify this in your core.js file, for example.  Then anywhere in
156   your application you can use it to define bindings like so:
157 
158         valueBinding: SC.Binding.from("MyApp.someController.value").notLessThan(10)
159 
160   Also, remember that helpers are chained so you can use your helper along with
161   any other helpers.  The example below will create a one way binding that
162   does not allow empty values or values less than 10:
163 
164         valueBinding: SC.Binding.oneWay("MyApp.someController.value").notEmpty().notLessThan(10)
165 
166   Note that the built in helper methods all allow you to pass a "from"
167   property path so you don't have to use the from() helper to set the path.
168   You can do the same thing with your own helper methods if you like, but it
169   is not required.
170 
171   Creating Custom Binding Templates
172   ===
173 
174   Another way you can customize bindings is to create a binding template.  A
175   template is simply a binding that is already partially or completely
176   configured.  You can specify this template anywhere in your app and then use
177   it instead of designating your own custom bindings.  This is a bit faster on
178   app startup but it is mostly useful in making your code less verbose.
179 
180   For example, let's say you will be frequently creating one way, not empty
181   bindings that allow values greater than 10 throughout your app.  You could
182   create a binding template in your core.js like this:
183 
184         MyApp.LimitBinding = SC.Binding.oneWay().notEmpty().notLessThan(10);
185 
186   Then anywhere you want to use this binding, just refer to the template like
187   so:
188 
189         valueBinding: MyApp.LimitBinding.beget("MyApp.someController.value")
190 
191   Note that when you use binding templates, it is very important that you
192   always start by using beget() to extend the template.  If you do not do
193   this, you will end up using the same binding instance throughout your app
194   which will lead to erratic behavior.
195 
196   How to Manually Activate a Binding
197   ===
198 
199   All of the examples above show you how to configure a custom binding, but
200   the result of these customizations will be a binding template, not a fully
201   active binding.  The binding will actually become active only when you
202   instantiate the object the binding belongs to.  It is useful however, to
203   understand what actually happens when the binding is activated. (Of course
204   you should always use the highest-level APIs available, even if you understand
205   how it works underneath; unless you have specific needs, you should rely on
206   the convenience `fooBinding` format.)
207 
208   For a binding to function it must have at least a "from" property and a "to"
209   property.  The from property path points to the object/key that you want to
210   bind from while the to path points to the object/key you want to bind to.
211 
212   When you define a custom binding, you are usually describing the property
213   you want to bind from (such as "MyApp.someController.value" in the examples
214   above).  When your object is created, it will automatically assign the value
215   you want to bind "to" based on the name of your binding key.  In the
216   examples above, during init, SproutCore objects will effectively call
217   something like this on your binding:
218 
219         binding = this.valueBinding.beget().to("value", this);
220 
221   This creates a new binding instance based on the template you provide, and
222   sets the to path to the "value" property of the new object.  Now that the
223   binding is fully configured with a "from" and a "to", it simply needs to be
224   connected to become active.  This is done through the connect() method:
225 
226         binding.connect();
227 
228   Now that the binding is connected, it will observe both the from and to side
229   and relay changes.
230 
231   If you ever needed to do so (you almost never will, but it is useful to
232   understand this anyway), you could manually create an active binding by
233   doing the following:
234 
235         SC.Binding.from("MyApp.someController.value")
236          .to("MyApp.anotherObject.value")
237          .connect();
238 
239   You could also use the bind() helper method provided by SC.Object. (This is
240   the same method used by SC.Object.init() to setup your bindings):
241 
242         MyApp.anotherObject.bind("value", "MyApp.someController.value");
243 
244   Both of these code fragments have the same effect as doing the most friendly
245   form of binding creation like so:
246 
247 
248         MyApp.anotherObject = SC.Object.create({
249           valueBinding: "MyApp.someController.value",
250 
251           // OTHER CODE FOR THIS OBJECT...
252 
253         });
254 
255   SproutCore's built in binding creation method make it easy to automatically
256   create bindings for you. If you need further documentation on SC.Binding's inner
257   workings, see the private method documentation in the source code.
258

268   /** @private
269     This is the core method you use to create a new binding instance.  The
270     binding instance will have the receiver instance as its parent which means
271     any configuration you have there will be inherited.
272 
273     The returned instance will also have its parentBinding property set to the
274     receiver.
275 
276     @param {String} [fromPath]
277     @returns {SC.Binding} new binding instance
278   */
279   beget: function (fromPath) {
280     var ret = SC.beget(this);
281     ret.parentBinding = this;
282     // Logic gates must be recreated on beget.
283     if (ret._LogicGate) {
284       ret._logicGate = ret._LogicGate.create(ret._logicGateHash);
285       ret = ret.from('logicProperty', ret._logicGate).oneWay();
286     }
287     // Enables duplicate API calls for SC.Binding.beget and SC.Binding.from
288     if (fromPath !== undefined) ret = ret.from(fromPath);
289     return ret;
290   },

293     Returns a builder function for compatibility.
294   */
295   builder: function () {
296     var binding = this,
297         ret = function (fromProperty) { return binding.beget().from(fromProperty); };
298     ret.beget = function () { return binding.beget(); };
299     return ret;

302   /**
303     This will set "from" property path to the specified value.  It will not
304     attempt to resolve this property path to an actual object/property tuple
305     until you connect the binding.
306 
307     The binding will search for the property path starting at the root level
308     unless you specify an alternate root object as the second parameter to this
309     method.  Alternatively, you can begin your property path with either "." or
310     "*", which will use the root object of the to side be default.  This special
311     behavior is used to support the high-level API provided by SC.Object.
312 
313     @param {String|Tuple} propertyPath A property path or tuple
314     @param {Object} [root] root object to use when resolving the path.
315     @returns {SC.Binding} this
316   */
317   from: function (propertyPath, root) {
318 
319     // if the propertyPath is null/undefined, return this.  This allows the
320     // method to be called from other methods when the fromPath might be
321     // optional. (cf single(), multiple())
322     if (!propertyPath) return this;
323 
324     // beget if needed.
325     var binding = (this === SC.Binding) ? this.beget() : this;
326     binding._fromPropertyPath = propertyPath;
327     binding._fromRoot = root;
328     binding._fromTuple = null;
329     return binding;
330   },
331 
332   /**
333     This will set the "to" property path to the specified value.  It will not
334     attempt to reoslve this property path to an actual object/property tuple
335     until you connect the binding.
336 
337     If you are using the convenience format `fooBinding`, for example
338     `isVisibleBinding`, you do not need to call this method, as the `to` property
339     path will be generated for you when its object is created.
340 
341     @param {String|Tuple} propertyPath A property path or tuple
342     @param {Object} [root] root object to use when resolving the path.
343     @returns {SC.Binding} this
344   */
345   to: function (propertyPath, root) {
346     // beget if needed.
347     var binding = (this === SC.Binding) ? this.beget() : this;
348     binding._toPropertyPath = propertyPath;
349     binding._toRoot = root;
350     binding._toTuple = null; // clear out any existing one.
351     return binding;
352   },
353 
354   /**
355     Attempts to connect this binding instance so that it can receive and relay
356     changes.  This method will raise an exception if you have not set the
357     from/to properties yet.
358 
359     @returns {SC.Binding} this
360   */
361   connect: function () {
362     // If the binding is already connected, do nothing.
363     if (this.isConnected) return this;
364     this.isConnected = YES;
365     this._connectionPending = YES; // its connected but not really...
366     this._syncOnConnect = YES;
367 
368     SC.Binding._connectQueue.add(this);
369 
370     if (!SC.RunLoop.isRunLoopInProgress()) {
371       this._scheduleSync();
372     }
373 
374     return this;
375   },
376 
377   /** @private
378     Actually connects the binding.  This is done at the end of the runloop
379     to give you time to setup your entire object graph before the bindings
380     try to activate.
381   */
382   _connect: function () {
383     if (!this._connectionPending) return; //nothing to do
384     this._connectionPending = NO;
385 
386     var path, root;
387 
388     //@if(debug)
389     var bench = SC.BENCHMARK_BINDING_SETUP;
390     if (bench) SC.Benchmark.start("SC.Binding.connect()");
391     //@endif
392 
393     // try to connect the from side.
394     // as a special behavior, if the from property path begins with either a
395     // . or * and the fromRoot is null, use the toRoot instead.  This allows
396     // for support for the SC.Object shorthand:
397     //
398     // contentBinding: "*owner.value"
399     //
400     path = this._fromPropertyPath;
401     root = this._fromRoot;
402 
403     if (typeof path === "string") {
404 
405       // if the first character is a '.', this is a static path.  make the
406       // toRoot the default root.
407       if (path.indexOf('.') === 0) {
408         path = path.slice(1);
409         if (!root) root = this._toRoot;
410 
411       // if the first character is a '*', then setup a tuple since this is a
412       // chained path.
413       } else if (path.indexOf('*') === 0) {
414         path = [this._fromRoot || this._toRoot, path.slice(1)];
415         root = null;
416       }
417     }
418     this._fromObserverData = [path, this, this.fromPropertyDidChange, root];
419     SC.Observers.addObserver.apply(SC.Observers, this._fromObserverData);
420 
421     // try to connect the to side
422     if (!this._oneWay) {
423       path = this._toPropertyPath;
424       root = this._toRoot;
425       this._toObserverData = [path, this, this.toPropertyDidChange, root];
426       SC.Observers.addObserver.apply(SC.Observers, this._toObserverData);
427     }
428 
429     //@if(debug)
430     if (bench) SC.Benchmark.end("SC.Binding.connect()");
431     //@endif
432 
433     // now try to sync if needed
434     if (this._syncOnConnect) {
435       this._syncOnConnect = NO;
436       //@if(debug)
437       if (bench) SC.Benchmark.start("SC.Binding.connect().sync");
438       //@endif
439       this.sync();
440       //@if(debug)
441       if (bench) SC.Benchmark.end("SC.Binding.connect().sync");
442       //@endif
443     }
444   },
445 
446   /** 
447     Disconnects the binding instance.  Changes will no longer be relayed.  You
448     will not usually need to call this method.
449 
450     @returns {SC.Binding} this
451   */
452   disconnect: function () {
453     if (!this.isConnected) return this; // nothing to do.
454 
455     // if connection is still pending, just cancel
456     if (this._connectionPending) {
457       this._connectionPending = NO;
458 
459       SC.Binding._connectQueue.remove(this);
460     // connection is completed, disconnect.
461     } else {
462       SC.Observers.removeObserver.apply(SC.Observers, this._fromObserverData);
463       if (!this._oneWay) {
464         SC.Observers.removeObserver.apply(SC.Observers, this._toObserverData);
465       }
466 
467       // Remove ourselves from the change queue (if we are in it).
468       SC.Binding._changeQueue.remove(this);
469     }
470 
471     this.isConnected = NO;
472     return this;
473   },

483   /** @private
484     Disconnects the binding and removes all properties and external references. Called by
485     either binding target object when destroyed.
486 
487     @private
488   */
489   destroy: function () {
490     // If we're already destroyed, there's nothing to do.
491     if (this.isDestroyed) return;
492 
493     // Mark it destroyed.
494     this.isDestroyed = YES;
495 
496     // Clean up the logic gate, if any. (See logic gate methods.)
497     if (this._logicGate) {
498       this._logicGate.destroy();
499       this._logicGate = null;
500       this._LogicGate = null;
501       this._logicGateHash = null;
502     }
503

505     this.disconnect();
506 
507     // Aggressively null out internal properties.
508     this._bindingSource = null;
509     this._toRoot = this._toTarget = null;
510     this._fromRoot = this._fromTarget = null;
511     this._toObserverData = this._fromObserverData = null;

514   /** @private
515     Invoked whenever the value of the "from" property changes.  This will mark
516     the binding as dirty if the value has changed.
517 
518     @param {Object} target The object that contains the key
519     @param {String} key The name of the property which changed
520   */
521   fromPropertyDidChange: function (target, key) {
522     var v = target ? target.get(key) : null;
523 
524     // In rare circumstances, getting a property can result in observers firing,
525     // which may in turn run code that disconnects the binding. The cause of
526     // this pattern has been difficult to determine and so until a concrete test
527     // scenario and a lower level fix can be found, show a warning and ignore
528     // the update.
529     if (!this.isConnected) {
530       //@if(debug)
531       SC.Logger.warn("Developer Warning: A binding attempted to update after it was disconnected. The update will be ignored for binding: %@".fmt(this._fromPropertyPath, this._fromTarget, this));
532       //@endif
533 
534       // Break early.
535       return;
536     }
537 
538     // if the new value is different from the current binding value, then
539     // schedule to register an update.
540     if (v !== this._bindingValue || key === '[]') {
541 
542       this._setBindingValue(target, key);
543       SC.Binding._changeQueue.add(this); // save for later.
544 
545       this._scheduleSync();
546     }
547   },
548 
549   /** @private
550     Invoked whenever the value of the "to" property changes.  This will mark the
551     binding as dirty only if:
552 
553     - the binding is not one way
554     - the value does not match the stored transformedBindingValue
555 
556     if the value does not match the transformedBindingValue, then it will
557     become the new bindingValue.
558 
559     @param {Object} target The object that contains the key
560     @param {String} key The name of the property which changed
561   */
562   toPropertyDidChange: function (target, key) {
563     if (this._oneWay) return; // nothing to do
564 
565     var v = target.get(key);
566 
567     // In rare circumstances, getting a property can result in observers firing,
568     // which may in turn run code that disconnects the binding. The cause of
569     // this pattern has been difficult to determine and so until a concrete test
570     // scenario and a lower level fix can be found, show a warning and ignore
571     // the update.
572     if (!this.isConnected) {
573       //@if(debug)
574       SC.Logger.warn("Developer Warning: A binding attempted to update after it was disconnected. The update will be ignored for binding: %@".fmt(this));
575       //@endif
576 
577       // Break early.
578       return;
579     }
580 
581     // if the new value is different from the current binding value, then
582     // schedule to register an update.
583     if (v !== this._transformedBindingValue) {
584       this._setBindingValue(target, key);
585       SC.Binding._changeQueue.add(this); // save for later.
586 
587       this._scheduleSync();
588     }
589   },

591   /** @private */
592   _scheduleSync: function () {
593     if (SC.RunLoop.isRunLoopInProgress() || SC.Binding._syncScheduled) { return; }
594     SC.Binding._syncScheduled = YES;
595     setTimeout(function () { SC.run(); SC.Binding._syncScheduled = NO; }, 1);
596   },

598   /** @private
599     Saves the source location for the binding value.  This will be used later
600     to actually update the binding value.
601   */
602   _setBindingValue: function (source, key) {
603     this._bindingSource = source;
604     this._bindingKey    = key;
605   },
606 
607   /** @private
608     Updates the binding value from the current binding source if needed.  This
609     should be called just before using this._bindingValue.
610   */
611   _computeBindingValue: function () {
612     var source = this._bindingSource,
613         key    = this._bindingKey,
614         v;
615 
616     this._bindingValue = v = (source ? source.getPath(key) : null);
617     this._transformedBindingValue = this._computeTransformedValue(v);
618   },

620   /** @private
621     Applies transforms to the value and returns the transfomed value.
622     @param {*} value Binding value to transform
623     @returns {*} Transformed value
624   */
625   _computeTransformedValue: function (value) {
626     var transforms = this._transforms,
627         idx,
628         len,
629         transform;
630 
631     if (transforms) {
632       len = transforms.length;
633       for (idx = 0; idx < len; idx++) {
634         transform = transforms[idx];
635         value = transform(value, this);
636       }
637     }
638 
639     // if error objects are not allowed, and the value is an error, then
640     // change it to null.
641     if (this._noError && SC.typeOf(value) === SC.T_ERROR) { value = null; }
642 
643     return value;
644   },

646   _connectQueue: SC.CoreSet.create(),
647   _alternateConnectQueue: SC.CoreSet.create(),
648   _changeQueue: SC.CoreSet.create(),
649   _alternateChangeQueue: SC.CoreSet.create(),
650 
651   /** @private
652     Call this method on SC.Binding to flush all bindings with changes pending.
653 
654     @returns {Boolean} YES if changes were flushed.
655   */
656   flushPendingChanges: function () {
657 
658     // don't allow flushing more than one at a time
659     if (this._isFlushing) return NO;
660     this._isFlushing = YES;
661     SC.Observers.suspendPropertyObserving();
662 
663     var didFlush = NO,
664         // connect any bindings
665         queue, binding;
666 
667     while ((queue = this._connectQueue).length > 0) {
668       this._connectQueue = this._alternateConnectQueue;
669       this._alternateConnectQueue = queue;
670       while ((binding = queue.pop())) { binding._connect(); }
671     }
672 
673     // loop through the changed queue...
674     while ((queue = this._changeQueue).length > 0) {
675       //@if(debug)
676       if (SC.LOG_BINDINGS) SC.Logger.log("Begin: Trigger changed bindings");
677       //@endif
678 
679       didFlush = YES;
680 
681       // first, swap the change queues.  This way any binding changes that
682       // happen while we flush the current queue can be queued up.
683       this._changeQueue = this._alternateChangeQueue;
684       this._alternateChangeQueue = queue;
685 
686       // next, apply any bindings in the current queue.  This may cause
687       // additional bindings to trigger, which will end up in the new active
688       // queue.
689       while ((binding = queue.pop())) { binding.applyBindingValue(); }
690 
691       // now loop back and see if there are additional changes pending in the
692       // active queue.  Repeat this until all bindings that need to trigger
693       // have triggered.
694       //@if(debug)
695       if (SC.LOG_BINDINGS) SC.Logger.log("End: Trigger changed bindings");
696       //@endif
697     }
698 
699     // clean up
700     this._isFlushing = NO;
701     SC.Observers.resumePropertyObserving();
702 
703     return didFlush;
704   },

706   /** @private
707     This method is called at the end of the Run Loop to relay the changed

754   /**
755     Calling this method on a binding will cause it to check the value of the
756     from side of the binding matches the current expected value of the
757     binding. If not, it will relay the change as if the from side's value has
758     just changed.
759 
760     This method is useful when you are dynamically connecting bindings to a
761     network of objects that may have already been initialized. Otherwise you
762     should not need to call this method.
763   */
764   sync: function () {
765     var target,
766         key,
767         v,
768         tv;
769 
770     // do nothing if not connected
771     if (!this.isConnected) return this;
772 
773     // connection is pending, just note that we should sync also
774     if (this._connectionPending) {
775       this._syncOnConnect = YES;
776 
777     // we are connected, go ahead and sync
778     } else {
779       this._computeBindingTargets();
780       target = this._fromTarget;
781       key = this._fromPropertyKey;
782       if (!target || !key) return this; // nothing to do
783 
784       // Let's check for whether target is a valid observable with getPath.
785       // Common cases might have it be a Window or a DOM object.
786       //
787       // If we have a target, it is ready, but if it is invalid, that is WRONG.
788       if (!target.isObservable) {
789         //@if(debug)
790         // Provide some developer support.
791         if (target === window) {
792           var msg = "Developer Warning: You are attempting to bind \"%{to_root}\"'s '%{to_property}' property to the non-observable 'window.%{key}'. It's likely that you've specified a local binding path without prepending a period. For example, you may have `%{to_property}Binding: '%{key}'` instead of `%{to_property}Binding: '.%{key}'`.";
793           msg = msg.fmt({
794             to_root: (this._toRoot || 'object').toString(),
795             to_property: this._toPropertyPath,
796             key: key
797           });
798           SC.Logger.warn(msg);
799         } else {
800           SC.Logger.warn("Developer Warning: Cannot bind \"%@\"'s '%@' property to property '%@' on non-observable '%@'".fmt((this._toRoot || 'object').toString(), this._toPropertyPath, key, target));
801         }
802         //@endif
803         return this;
804       }
805 
806       // get the new value
807       v = target.getPath(key);
808       tv = this._computeTransformedValue(v);
809 
810       // if the new value is different from the current binding value, then
811       // schedule to register an update.
812       if (v !== this._bindingValue || tv !== this._transformedBindingValue || key === '[]') {
813         this._setBindingValue(target, key);
814         SC.Binding._changeQueue.add(this); // save for later.
815       }
816     }
817 
818     return this;
819   },

826   /** @private */
827   _computeBindingTargets: function () {
828     var path, root, tuple;
829 
830     if (!this._fromTarget) {
831       // if the fromPropertyPath begins with a . or * then we may use the
832       // toRoot as the root object.  Similar code exists in connect() so if
833       // you make a change to one be sure to update the other.
834       path = this._fromPropertyPath;
835       root = this._fromRoot;
836       if (typeof path === "string") {
837 
838         // static path beginning with the toRoot
839         if (path.indexOf('.') === 0) {
840           path = path.slice(1); // remove the .
841           if (!root) root = this._toRoot; // use the toRoot optionally
842 
843         // chained path beginning with toRoot.  Setup a tuple
844         } else if (path.indexOf('*') === 0) {
845           path = [root || this._toRoot, path.slice(1)];
846           root = null;
847         }
848       }
849 
850       tuple = SC.tupleForPropertyPath(path, root);
851       if (tuple) {
852         this._fromTarget = tuple[0];
853         this._fromPropertyKey = tuple[1];
854       }
855     }
856 
857     if (!this._toTarget) {
858       path = this._toPropertyPath;
859       root = this._toRoot;
860       tuple = SC.tupleForPropertyPath(path, root);
861       if (tuple) {
862         this._toTarget = tuple[0];
863         this._toPropertyKey = tuple[1];
864         // Hook up _logicGate if needed (see logic gate methods).
865         if (this._logicGate) {
866           this._logicGate.set('localObject', this._toTarget);
867         }
868       }
869     }
870   },

876   /**
877     Configures the binding as one way.  A one-way binding will relay changes
878     on the "from" side to the "to" side, but not the other way around.  This
879     means that if you change the "to" side directly, the "from" side will not
880     be updated, and may have a different value.
881 
882     @param {String} [fromPath] from path to connect.
883     @param {Boolean} [aFlag] Pass NO to set the binding back to two-way
884     @returns {SC.Binding} this
885   */
886   oneWay: function (fromPath, aFlag) {
887 
888     // If fromPath is a bool but aFlag is undefined, swap.
889     if ((aFlag === undefined) && (SC.typeOf(fromPath) === SC.T_BOOL)) {
890       aFlag = fromPath;
891       fromPath = null;
892     }
893 
894     // beget if needed.
895     var binding = this.from(fromPath);
896     if (binding === SC.Binding) binding = binding.beget();
897     binding._oneWay = (aFlag === undefined) ? YES : aFlag;
898 
899     return binding;
900   },

902   /**
903     Adds the specified transform function to the array of transform functions.
904 
905     The function you pass must have the following signature:
906 
907           function (value) {};
908 
909     or:
910 
911           function (value, binding) {};
912 
913     It must return either the transformed value or an error object.
914 
915     Transform functions are chained, so they are called in order.  If you are
916     extending a binding and want to reset its transforms, you can call
917     resetTransform() first.
918 
919     @param {Function} transformFunc the transform function.
920     @returns {SC.Binding} this
921   */
922   transform: function (transformFunc) {
923     var binding = (this === SC.Binding) ? this.beget() : this;
924     var t = binding._transforms;
925 
926     // clone the transform array if this comes from the parent
927     if (t && (t === binding.parentBinding._transforms)) {
928       t = binding._transforms = t.slice();
929     }
930 
931     // create the transform array if needed.
932     if (!t) t = binding._transforms = [];
933 
934     // add the transform function
935     t.push(transformFunc);
936     return binding;
937   },

940     Resets the transforms for the binding.  After calling this method the
941     binding will no longer transform values.  You can then add new transforms
942     as needed.
943 
944     @returns {SC.Binding} this
945   */
946   resetTransforms: function () {
947     var binding = (this === SC.Binding) ? this.beget() : this;
948     binding._transforms = null;
949     return binding;

979   /**
980     Adds a transform to the chain that will allow only single values to pass.
981     This will allow single values, nulls, and error values to pass through.  If
982     you pass an array, it will be mapped as so:
983 
984           [] => null
985           [a] => a
986           [a,b,c] => Multiple Placeholder
987 
988     You can pass in an optional multiple placeholder or it will use the
989     default.
990 
991     Note that this transform will only happen on forwarded valued.  Reverse
992     values are send unchanged.
993 
994     @param {String} fromPath from path or null
995     @param {Object} [placeholder] placeholder value.
996     @returns {SC.Binding} this
997   */
998   single: function (fromPath, placeholder) {
999     if (placeholder === undefined) {
1000       placeholder = SC.MULTIPLE_PLACEHOLDER;
1001     }
1002     return this.from(fromPath).transform(function (value, isForward) {
1003       if (value && value.isEnumerable) {
1004         var len = value.get('length');
1005         value = (len > 1) ? placeholder : (len <= 0) ? null : value.firstObject();
1006       }
1007       return value;
1008     });

1118   /* @private Used by logic gate bindings. */
1119   _logicGateBinding: function (gateClass, pathA, pathB) {
1120     // If either path is local, remove any * chains and append the localObject path to it.
1121     if (pathA.indexOf('*') === 0 || pathA.indexOf('.') === 0) {
1122       pathA = pathA.slice(1).replace(/\*/g, '.');
1123       pathA = '*localObject.' + pathA;
1124     }
1125     if (pathB.indexOf('*') === 0 || pathB.indexOf('.') === 0) {
1126       pathB = pathB.slice(1).replace(/\*/g, '.');
1127       pathB = '*localObject.' + pathB;
1128     }
1129 
1130     // Gets the gate class and instantiates a nice copy.
1131     var gateHash = {
1132         localObject: null,
1133         valueABinding: SC.Binding.oneWay(pathA),
1134         valueBBinding: SC.Binding.oneWay(pathB)
1135       },
1136       gate = gateClass.create(gateHash);
1137 
1138     // Creates and populates the return binding.
1139     var ret = this.from('logicProperty', gate).oneWay();
1140     // This is all needed later on by beget, which must create a new logic gate instance
1141     // or risk bad behavior.
1142     ret._LogicGate = gateClass;
1143     ret._logicGateHash = gateHash;
1144     ret._logicGate = gate;
1145 
1146     // On our way.
1147     return ret;
1148   },
